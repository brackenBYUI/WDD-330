At work we program larger to smaller I really liked the quote by Joel Marsh "If you design for the smallest, least powerful device first, then you will focus on the content, and your core functionality. That leads to simple, beautiful apps."
If we have to change our minds now for the new viewing devices that means we will continue to change our minds because new viewing devices are being released all the time.
Mobile devices are changed out way more ofter than desktops. Desktops replaced every 4-5 years. Mobile devices replaced every ~2 years.
Phones are more widely available to more people. Most home have a phone for each person but same home will share the same desktop
Downsides to mobile first. Time consuming and costly for a full rebuild.
Make things as easy as possible for users. When having a user complete a task make it as few of steps as possible.
Make your page as efficiency as possible. Speed matters don't make users wait.

Optimize Site/App

  • Reduce image usage throughout sites/apps
  • Optimize and minify CSS and JS files
  • GZIP files to eliminate unnecessary data.

Move most commonly used items to the bottom of the page. Think about having the main nav to the bottom of the page.
Think about removing the Hamburger menu button and just use Menu with a boarder.
Remove labels on Forms for mobile devices will make it look a lot better.
Also use float labels so you know what the field is after filling in information on the form
Enable auto complete on forms to help users fill out form quicker and with less errors.
Try using an single line input field for credit card info.
Make sure to use the correct keyboard for the input
Don't use pop up adds ever.
Make adds small and simple. Should not take 40% or more of window.
Have adds reveal something when the user clicks on them.
"the fallen state of our Internet is a direct, if unintentional, consequence of choosing advertising as the default model to support online content and services." - Ethan Zuckerman
